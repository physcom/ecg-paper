The core change

The Core Change: Anti-Aliased Decimation

A three-line patch to preprocessing. Nothing else changes.

preprocessing.py · step 6 
from scipy.signal import decimate

# factor = 5   →  5000 → 1000 samples  (100 Hz effective)
# factor = 10  →  5000 →  500 samples  ( 50 Hz effective)
x_down = decimate(x, factor,
                  ftype='iir',     # Chebyshev type-I low-pass
                  n=8,             # filter order
                  zero_phase=True) # preserves P-wave phase
scipy.signal.decimate — applied once at ingest, before the model ever sees the signal.

Phase preserved

zero_phase=True runs the filter forward and backward, so P-wave and QRS timing are not shifted.

Proper anti-aliasing

Chebyshev type-I low-pass at the new Nyquist removes any signal that would otherwise fold into the QRS band — the difference between +10 pp F1 and worse than baseline.

Off-line, no training cost

Applied once at ingest. Zero training overhead, 3.3× faster inference, smaller tensors.

Same data hygiene as baseline

Seed, class weights, augmentation recipe and the split are unchanged from the 5000-sample baseline — the gain is attributable to this step alone.

Why the override path is a first-class button.

  • The doctor's name goes on the report — not the model's.
  • Overrides are feedback signal: the analytics tab measures per-clinician agreement rate, and the model retraining pipeline consumes high-confidence overrides as corrective labels.

Safety & governance

Four guard-rails, visible to the doctor at all times.

1 · Confidence gate

Cases under 85% AI confidence never auto-close. The clinician must confirm or override.

2 · Override is free

No "Are you sure?" modal. One click, one required reason, done.

3 · Full audit trail

Every interaction timestamped, attributed, and immutable. Exportable by compliance.

4 · No PHI leaves the hospital

All inference runs on an on-prem GPU. LLM narration uses a local model; no external API is called.

Known weak spots, surfaced

Brugada, long-QT, paediatric ECGs — the tool tells the doctor "out of training distribution" instead of guessing.

Clear role boundary

CardioLens is decision support, not a medical device. Final diagnosis is the clinician's; report is signed by them.

How it sits in the hospital

No new logins. No changes to the EHR.

Deploy

One Docker compose, one on-prem GPU (e.g. RTX 5090 / A4000). MRN comes from the EHR; no new patient master.

Authenticate

SAML/OIDC against the hospital IdP. SSO means no new passwords for the clinician.

Integrate

HL7 aECG ingest, FHIR DiagnosticReport out. Falls back to PDF-to-folder for EHRs without an API.
